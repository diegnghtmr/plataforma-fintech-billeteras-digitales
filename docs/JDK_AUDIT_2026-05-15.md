## Classification Notes

### Category A — Legitimate Boundary Translation (ADR-9.1)

All 22 hits fall here. The pattern is uniform:

```java
// Internal accumulator built from domain types
List<MetricItem> result = new ArrayList<>();
for (DomainEntity entity : domainPort.findAll()) {
    result.add(new MetricItem(...));
}
return result;  // consumed by Spring REST controller
```

Per ADR-9.1, use cases are allowed to return `java.util.List<ViewRecord>` at the output edge because:
1. The view records (e.g. `MetricItem`, `RankingItem`) are application-layer types, not domain objects.
2. Spring's Jackson serializer requires standard JDK collections to serialize JSON arrays.
3. The domain ports themselves return domain-owned types (`MiLista`, `Iterable`, `Optional`).

### Category B — Internal Storage Violations

**Count: 0**

No use of `java.util.HashMap`, `HashSet`, `TreeMap`, `TreeSet`, `LinkedList` as internal storage was found. All internal aggregation uses `TablaHash`, `ArbolBST`, `MiLista`, and `ColaSimple`.

### Category C — DTO/Result Record Fields

**Count: 0**

`application/result/ExecutionReport.java` was verified to use `MiLista<String>` for its list fields (not `java.util.List`). All other result records use primitive types or domain enums only.

## ADR-9.3 — `domain.structures` boundary helpers

`domain.structures` contains the project's own data structures (`MiLista`, `TablaHash`, `Conjunto`, `ArbolBST`, `GrafoTransferencias`, `Pila`, `ColaSimple`, `ColaPrioridad`). A handful of these expose **boundary helper methods** that return `java.util.List` so consumers (tests, infrastructure adapters, port impls) can iterate without depending on the internal node types:

- `MiLista.toList()` — read-only snapshot for adapters/tests.
- `TablaHash.entries()/keys()/values()` — flatten internal buckets.
- `ArbolBST.toList()` — in-order traversal snapshot.
- `GrafoTransferencias.findCycles()` — returns `List<List<String>>` of normalized cycles (the port `TransferGraphRepository.findCycles()` wraps this into `MiLista<MiLista<String>>` at the adapter).

These return-type uses are scoped strictly to **boundary translation** (snapshot/serialization), never to internal storage. Internal node arrays and chaining within each structure remain pure (no `HashMap`, `HashSet`, `LinkedList`, etc.).

ArchUnit rules 1–22 deliberately do **not** apply to `..domain.structures..` because the layer would be unable to bridge to standard Java without these helpers. The exclusion is intentional, documented here, and parallel in spirit to ADR-9.1 (Jackson boundary at the application return edge).

## ADR-9.2 — `java.util.Optional` is a structural primitive

`java.util.Optional` appears in domain ports (`UserRepository.findByEmail`, `WalletRepository.findById`, etc.) and a handful of use cases. It is **not** classified as a collection and is **not** banned by the academic rule in `INFORME_TECNICO.md:7`.

Rationale:

1. `Optional<T>` is a single-element value wrapper, not a data structure for aggregation.
2. It is analogous to `java.time.Instant` — a structural type used to express domain semantics (presence/absence) without leaking storage choices.
3. Re-implementing `Optional` would force every port consumer to learn a custom equivalent for zero academic gain.
4. ArchUnit rules in `HexagonalBoundaryTest.java` explicitly do **not** ban `java.util.Optional` in any package.

This carve-out is parallel to ADR-9.1 (boundary translation for collections at the REST edge).

## Recommendations

1. **No immediate action required** — zero class B violations and zero internal `java.util` storage inside `application.usecase` remain after Round 2.
2. **ArchUnit boundary test** (`HexagonalBoundaryTest.java`) now enforces:
   - Domain ports do not use any of the 9 forbidden JDK collection types.
   - `domain.model` and `domain.service` do not use `HashMap`, `HashSet`, `TreeMap`, `TreeSet`, `LinkedList`, `List`, or `ArrayList`.
   - `application.usecase` does not use `HashMap`, `HashSet`, `TreeMap`, `TreeSet`, or `LinkedList`.
3. **Boundary-only carve-out**: `java.util.List` / `java.util.ArrayList` remain accepted in `application.usecase` **as REST return types only** (Jackson requirement). Internal accumulation must use `MiLista`.
